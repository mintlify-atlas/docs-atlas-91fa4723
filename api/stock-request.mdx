## Overview

The `stock.request` model represents individual stock requests in the Odoo Stock Request framework.

- **Model name**: `stock.request`
- **Description**: Stock Request
- **Inherits from**: `stock.request.abstract`
- **Order**: `id desc`

## Purpose

This model manages stock requests that trigger procurement rules to fulfill inventory needs. It tracks the lifecycle of requests from draft through confirmation, execution, and completion, with support for allocations, stock moves, and pickings.

## Methods

### action_confirm()

Confirms the stock request and triggers procurement rule execution.

**Returns:** `bool` - Always returns `True`

**State Transition:** `draft` → `open` (or `done` if immediately fulfilled)

```python
def action_confirm(self):
    self._action_confirm()
    return True
```

**Internal implementation:**

```python
def _action_confirm(self):
    self._action_launch_procurement_rule()
    self.filtered(lambda x: x.state != "done").write({"state": "open"})
```

**Usage:**

```python
# Confirm a single request
request = env['stock.request'].browse(request_id)
request.action_confirm()

# Confirm multiple requests
requests = env['stock.request'].browse([id1, id2, id3])
requests.action_confirm()
```

### _prepare_procurement_values()

Prepares procurement values for creating stock moves or purchase orders.

**Parameters:**
- `group_id` (int, optional): Procurement group ID. If not provided, uses `self.procurement_group_id`

**Returns:** `dict` - Dictionary with procurement values

```python
def _prepare_procurement_values(self, group_id=False):
    return {
        "date_planned": self.expected_date,
        "warehouse_id": self.warehouse_id,
        "stock_request_allocation_ids": self.id,
        "group_id": group_id or self.procurement_group_id.id or False,
        "route_ids": self.route_id,
        "stock_request_id": self.id,
    }
```

**Usage:**

```python
request = env['stock.request'].browse(request_id)
values = request._prepare_procurement_values()
print(values)
# Output: {'date_planned': datetime, 'warehouse_id': warehouse, ...}
```

**Extension:**

This method can be overridden to add custom keys for procurement:

```python
def _prepare_procurement_values(self, group_id=False):
    values = super()._prepare_procurement_values(group_id)
    values.update({
        'custom_field': self.custom_field,
        'priority': self.priority,
    })
    return values
```

### _action_launch_procurement_rule()

Launches procurement rules to fulfill the stock request. Executes `_run_move`, `_run_buy`, or `_run_manufacture` depending on product configuration.

**Returns:** `bool` - Always returns `True`

**Raises:** `UserError` - If procurement fails

```python
def _action_launch_procurement_rule(self):
    precision = self.env["decimal.precision"].precision_get(
        "Product Unit of Measure"
    )
    errors = []
    for request in self:
        if request._skip_procurement():
            continue
        qty = 0.0
        for move in request.move_ids.filtered(lambda r: r.state != "cancel"):
            qty += move.product_qty

        if float_compare(qty, request.product_qty, precision_digits=precision) >= 0:
            continue

        # If stock is available we use it and we do not execute rule
        if request.company_id.stock_request_check_available_first:
            if (
                float_compare(
                    request.product_id.sudo()
                    .with_context(location=request.location_id.id)
                    .free_qty,
                    request.product_uom_qty,
                    precision_digits=precision,
                )
                >= 0
            ):
                request._action_use_stock_available()
                continue

        values = request._prepare_procurement_values(
            group_id=request.procurement_group_id
        )
        try:
            procurements = []
            procurements.append(
                self.env["procurement.group"].Procurement(
                    request.product_id,
                    request.product_uom_qty,
                    request.product_uom_id,
                    request.location_id,
                    request.name,
                    request.name,
                    self.env.company,
                    values,
                )
            )
            self.env["procurement.group"].run(procurements)
        except UserError as error:
            errors.append(str(error))
    if errors:
        raise UserError("\n".join(errors))
    return True
```

**Behavior:**

1. Skips procurement if state is not `draft` or product type is not stockable
2. Checks if quantity is already satisfied by existing moves
3. If company setting enabled, attempts to use available stock first
4. Otherwise, creates procurement to trigger buy/manufacture/move rules

**Usage:**

```python
request = env['stock.request'].browse(request_id)
try:
    request._action_launch_procurement_rule()
except UserError as e:
    print(f"Procurement failed: {e}")
```

### check_done()

Checks if the request should be marked as done based on allocated quantities.

**Returns:** `bool` - Always returns `True`

```python
def check_done(self):
    precision = self.env["decimal.precision"].precision_get(
        "Product Unit of Measure"
    )
    for request in self:
        allocated_qty = sum(request.allocation_ids.mapped("allocated_product_qty"))
        qty_done = request.product_id.uom_id._compute_quantity(
            allocated_qty, request.product_uom_id
        )
        if (
            float_compare(
                qty_done, request.product_uom_qty, precision_digits=precision
            )
            >= 0
        ):
            request.action_done()
        elif request._check_cancel_allocation():
            # If qty_done=0 and qty_cancelled>0 it's cancelled
            request.write({"state": "cancel"})
    return True
```

### check_cancel()

Checks if the request should be marked as cancelled based on allocations.

def check_cancel(self):
    for request in self:
        if request._check_cancel_allocation():
            request.write({"state": "cancel"})

### action_view_transfer()

Opens the view showing associated delivery orders/pickings.

**Returns:** `dict` - Action definition for opening picking tree/form view

```python
def action_view_transfer(self):
    action = self.env["ir.actions.act_window"]._for_xml_id(
        "stock.action_picking_tree_all"
    )
    pickings = self.mapped("picking_ids")
    if len(pickings) > 1:
        action["domain"] = [("id", "in", pickings.ids)]
    elif pickings:
        action["views"] = [(self.env.ref("stock.view_picking_form").id, "form")]
        action["res_id"] = pickings.id
    return action
```

**Usage:**

```python
request = env['stock.request'].browse(request_id)
action = request.action_view_transfer()
# Returns action to open picking view(s)
```

## Lifecycle

```python
# Create a stock request
request = env['stock.request'].create({
    'product_id': product.id,
    'product_uom_qty': 10.0,
    'product_uom_id': product.uom_id.id,
    'warehouse_id': warehouse.id,
    'location_id': location.id,
    'requested_by': env.user.id,
})
# State: draft

# Confirm the request
request.action_confirm()
# State: open
# Procurement rules triggered

# System automatically updates quantities via allocations
# qty_in_progress and qty_done are computed

# When fulfilled, request is marked done
request.check_done()
# State: done
```
