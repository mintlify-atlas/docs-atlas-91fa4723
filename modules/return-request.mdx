The Stock Return Request module enables returning stock from a location to suppliers, customers, or internal locations by reverting the corresponding pickings. This provides a structured way to handle returns, defects, and inventory corrections.

## Overview

This module allows comprehensive return management for various scenarios: returning defective products to suppliers, processing customer returns, or transferring stock between internal locations. It reverses the original picking operations while maintaining traceability.

## Key Features

- **Multiple Return Types**: Return to suppliers, customers, or internal locations
- **Picking Reversal**: Automatically reverses original picking operations
- **Return Tracking**: Complete traceability of return operations
- **Quantity Management**: Track returnable quantities on stock moves
- **Return Suggestions**: Wizard to suggest returnable items
- **Return Reports**: Printable return documentation
- **Integration**: Works with standard Odoo inventory workflows

## Usage

### Creating a Return Request

1. Navigate to **Inventory > Operations > Return Request**
2. Click **Create**
3. Select the return type:
   - Return to Supplier
   - Return from Customer
   - Internal Location Return
4. Fill in the required information
5. Select products and quantities to return
6. Click **Confirm**

### Return to Supplier

**Use Case**: Return defective or excess items to a supplier

1. Go to **Inventory > Operations > Return Request**
2. Click **Create**
3. Select **Return to Supplier**
4. Choose the supplier
5. Select items to return:
   - Can select from received items
   - System shows returnable quantity
6. Enter reason for return
7. Click **Confirm**
8. Process the return picking

<Info>
The system creates a return picking that reverses the original receipt.
</Info>

### Return from Customer

**Use Case**: Process customer returns (defects, wrong items, etc.)

1. Go to **Inventory > Operations > Return Request**
2. Click **Create**
3. Select **Return from Customer**
4. Choose the customer
5. Optionally link to original delivery
6. Add products being returned
7. Specify return reason
8. Click **Confirm**
9. Receive the returned goods

### Internal Location Return

**Use Case**: Move stock back to a previous location

1. Go to **Inventory > Operations > Return Request**
2. Click **Create**
3. Select **Internal Return**
4. Choose source and destination locations
5. Select products to return
6. Specify quantities
7. Click **Confirm**
8. Process the internal transfer

## Returnable Quantity

The module adds a `qty_returnable` field to stock moves:

- **Calculation**: Based on original move quantity minus already returned
- **Updates**: Recalculated when returns are processed
- **Visibility**: Shows in move details
- **Validation**: System prevents over-returns

<Warning>
You cannot return more than the returnable quantity for a move.
</Warning>

## Return Suggestion Wizard

For easier return creation:

1. Open a completed picking/delivery
2. Click **Suggest Return**
3. System shows returnable items from that picking
4. Select items to return
5. Adjust quantities if needed
6. Click **Create Return Request**

<Info>
The wizard pre-fills all information from the original picking.
</Info>

### Models Added

#### stock.return.request

Main model for return requests:

- `partner_id` - Supplier or customer
- `picking_id` - Original picking to reverse
- `location_id` - Source location
- `location_dest_id` - Destination location
- `line_ids` - Return request lines
- `state` - Draft, Confirmed, Done, Cancelled
- `return_picking_ids` - Generated return pickings

#### stock.return.request.line

Return line items:

- `product_id` - Product to return
- `quantity` - Return quantity
- `uom_id` - Unit of measure
- `move_id` - Original stock move
- `reason` - Return reason

### Fields Added to Stock Move

- `qty_returnable` - Computed field for returnable quantity
- `qty_returned` - Total quantity already returned
- `return_move_ids` - One2many to return moves

- Return request form
- Return delivery slip
- Return receipt

## Known Issues / Roadmap

## Workflow Examples

### Defective Product Return

```
Receive Goods → Quality Check → Found Defective → Create Return Request → Ship Back to Supplier
```

1. Receive products from supplier
2. Quality control finds defects
3. Create return request for defective items
4. System generates return picking
5. Ship back to supplier
6. Track return status

### Customer Return Processing

```
Customer Request → Authorize Return → Receive Item → Inspect → Refund/Replace
```

1. Customer contacts about return
2. Create return request
3. Provide return authorization to customer
4. Receive returned item
5. Inspect condition
6. Process refund or replacement

### Inventory Correction

```
Wrong Location → Create Internal Return → Transfer Back → Correct Location
```

1. Discover stock in wrong location
2. Create internal return request
3. Generate transfer
4. Move stock back to correct location
5. Update inventory records

## Troubleshooting

### Cannot Create Return

Common issues:

- Original picking not found
- No returnable quantity available
- Already fully returned
- Picking not in Done state

### Returnable Quantity is Zero

Possible causes:

- Items already fully returned
- Original move was cancelled
- Return period expired (if configured)
- Product configuration issues

### Return Picking Not Created

Check:

- Return request is confirmed
- Quantities are valid
- Locations are accessible
- Routes are configured
- User has appropriate rights
