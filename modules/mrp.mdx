The Stock Request MRP module integrates stock requests with manufacturing, allowing users to view and manage manufacturing orders (MOs) that are automatically created when stock requests are confirmed.

## Overview

This module enables users to display Manufacturing Orders that have been created as a consequence of Stock Requests. When a stock request is confirmed and procurement rules determine that manufacturing is required, the system generates the corresponding MO and provides direct access from the stock request.

## Key Features

- **Manufacturing Order Display**: View MOs generated from stock requests
- **Direct Navigation**: Access manufacturing orders from stock request forms
- **Automatic Integration**: Seamless connection with MRP system
- **Production Tracking**: Monitor manufacturing progress from stock request context
- **Smart Buttons**: Intuitive UI elements for accessing related MOs

## Installation

### Automatic Installation

The module installs automatically when:

1. `stock_request` module is installed
2. `mrp` (Manufacturing) module is installed
3. You update the module list or restart Odoo

### Manual Installation

If needed, manually install:

1. Navigate to **Apps**
2. Search for "Stock Request MRP"
3. Click **Install**

## Configuration

No specific configuration required. The module works automatically with your existing:

- Bill of Materials (BOMs)
- Manufacturing routes
- Procurement rules
- Work centers and routings

<Note>
Ensure your products have BOMs defined and manufacturing routes configured for the module to generate MOs.
</Note>

## Usage

### Creating Stock Requests that Trigger Manufacturing

1. Go to **Stock Requests > Stock Requests**
2. Click **Create**
3. Select a manufactured product (one with a BOM)
4. Enter the quantity and destination location
5. Click **Confirm**

### Viewing Manufacturing Orders

Once the stock request is confirmed:

1. Open the confirmed stock request
2. Look for the **Manufacturing Orders** smart button
3. Click to view all related manufacturing orders
4. Manage production directly from the MO or navigate to Manufacturing app

<Info>
The Manufacturing Orders button appears when the confirmation of the stock request results in an immediate manufacturing order.
</Info>

### Complete Workflow

```
Stock Request → Confirm → Procurement → Manufacturing Order → Production → Done
```

1. **Create Stock Request**
   - Product: Finished Good A
   - Quantity: 50 units
   - Location: WH/Stock

2. **System Generates MO**
   - Evaluates procurement rules
   - Finds manufacturing route
   - Creates Manufacturing Order with BOM components

3. **View from Stock Request**
   - Click MO smart button
   - Review components, work orders
   - Plan production

4. **Complete Production**
   - Process work orders
   - Consume components
   - Produce finished goods
   - Stock request is fulfilled

## Integration Scenarios

### Scenario 1: Simple Manufacturing

```
Stock Request → MO Created → Produce → Fulfill Request
```

- Single-level BOM
- All components available
- Direct production fulfills request

### Scenario 2: Multi-Level BOM

```
Stock Request → MO for Product → Sub-MOs for Components → Production Chain
```

- Complex product structure
- Multiple manufacturing orders created
- All visible from original stock request

### Scenario 3: Mixed Procurement

```
Stock Request → MO + Purchase Orders → Combined Fulfillment
```

- Some components manufactured
- Other components purchased
- Both visible from stock request (with purchase module)

## Troubleshooting

### MO Not Created

<Warning>
If no manufacturing order is created, verify:
</Warning>

- Product has an active BOM
- Manufacturing route is configured
- Procurement rules point to production
- Product is set to "Can be manufactured"

### Cannot View MOs

Ensure:

- You have access rights to Manufacturing app
- MOs were created (check in Manufacturing menu)
- Module is installed and updated
