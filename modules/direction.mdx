The Stock Request Direction module simplifies stock requests by allowing users to specify a direction (inbound or outbound) and warehouse instead of specific inventory locations. This makes requests more intuitive and accessible to users unfamiliar with detailed warehouse structures.

## Overview

This module allows users to request products to be transferred from or to warehouses by specifying the direction, eliminating the need to manually select specific inventory locations. The warehouse team can later define the appropriate routes and locations.

## Key Features

- **Direction Field**: Choose between "To Warehouse" or "From Warehouse"
- **Simplified Location Selection**: Select warehouse instead of specific location
- **Automatic Location Resolution**: System determines appropriate locations based on direction
- **User-Friendly Interface**: Reduces complexity for end users
- **Warehouse Team Review**: Allows warehouse team to define final routes
- **Expected Date Support**: Plan requests with delivery expectations

1. Ensure `stock_request` module is installed
2. Install `stock_request_direction` from Apps menu

## Configuration

### Settings Configuration

The module adds direction-related settings:

1. Go to **Inventory > Configuration > Settings**
2. Look for Stock Request Direction settings
3. Configure default behaviors if needed

### Warehouse Configuration

Ensure your warehouses are properly configured:

- Warehouse locations defined
- Input/Output locations set
- Routes configured appropriately

<Info>
No complex configuration is required. The module uses standard warehouse configuration.
</Info>

## Usage

### Creating a Directional Stock Request

1. Navigate to **Stock Requests > Stock Requests**
2. Click **Create**
3. Fill in the request details:
   - **Product**: Select the product to request
   - **Quantity**: Enter the requested quantity
   - **Direction**: Choose "To Warehouse" or "From Warehouse"
   - **Warehouse**: Select the warehouse (instead of specific location)
   - **Expected Date**: Set when you need the stock
4. Click **Confirm**

<Info>
Upon confirmation, the warehouse team can review and define the appropriate route and specific locations.
</Info>

### Direction Options

#### To Warehouse (Inbound)

Request products to be delivered to a warehouse:

- **Use Case**: Need materials at a specific warehouse
- **Example**: "I need 100 units of Product A at Warehouse 1"
- **System Behavior**: Routes to warehouse input location

#### From Warehouse (Outbound)

Request products to be sent from a warehouse:

- **Use Case**: Need to send materials from a warehouse
- **Example**: "Send 50 units of Product B from Warehouse 2"
- **System Behavior**: Sources from warehouse stock locations

### Complete Workflow

1. **User Creates Request**
   - Selects product and quantity
   - Chooses direction and warehouse
   - Sets expected date
   - Clicks Confirm

2. **Warehouse Team Reviews**
   - Receives notification of new request
   - Reviews direction and requirements
   - Defines specific route if needed
   - Approves routing

3. **System Creates Transfers**
   - Generates appropriate stock moves
   - Creates pickings based on direction
   - Assigns to warehouse operations

4. **Execution**
   - Warehouse processes the transfer
   - User can track from "Transfers" button
   - Request marked complete when done

## Use Cases

### Inter-Warehouse Transfers

**Scenario**: Request stock from another warehouse

```
Warehouse A needs materials from Warehouse B:
- Direction: From Warehouse
- Warehouse: Warehouse B
- Product: Component X
- Quantity: 100
```

### Vendor Returns

**Scenario**: Return defective items to supplier

```
Return items from quality check:
- Direction: From Warehouse
- Warehouse: QC Warehouse
- Product: Defective Part Y
- Quantity: 25
```

### Customer Deliveries

**Scenario**: Request stock for customer delivery

```
Prepare customer shipment:
- Direction: From Warehouse
- Warehouse: Main Warehouse
- Product: Finished Good Z
- Quantity: 50

### Production Materials

```
Get materials to production floor:
- Direction: To Warehouse
- Warehouse: Production Warehouse
- Product: Raw Material A

## Best Practices

### For End Users

- Always specify expected dates for better planning
- Choose the correct direction carefully
- Provide notes for special handling requirements
- Select the appropriate warehouse based on need

### For Warehouse Teams

- Review directional requests promptly
- Verify routing is optimal before approval
- Check stock availability for from-warehouse requests

### For Administrators

## Advantages

## Cancellation

<Warning>
When cancelling a directional stock request, related pending stock moves will also be cancelled.
</Warning>

The cancellation process:

1. Open the stock request
2. Click **Cancel**
3. Confirm cancellation
4. System cancels related pending moves
5. Completed transfers remain unchanged
