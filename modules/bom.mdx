The Stock Request BOM module enhances stock request functionality by integrating with Bill of Materials (BOM), allowing users to automatically populate stock request lines based on a selected BOM and quantity.

## Overview

This module streamlines the process of requesting all components needed for a product. Instead of manually adding each component, users can select a BOM and quantity, and the system automatically creates stock request lines for all required materials.

## Key Features

- **BOM Integration**: Select a product BOM directly in stock request orders
- **Auto-Fill Components**: Automatically create request lines from BOM structure
- **Quantity Calculation**: Adjust all component quantities based on BOM quantity
- **Update Capability**: Modify BOM quantity and update all lines accordingly
- **Clear Function**: Remove BOM selection and related lines
- **Multi-Level Support**: Handle complex BOM structures

## Usage

### Creating a Stock Request Order with BOM

1. Navigate to **Stock Requests > Stock Request Orders**
2. Click **Create**
3. Fill in the header information:
   - **Product BOM**: Select a product that has a BOM defined
   - **Quantity BOM**: Enter the quantity of the finished product
4. The system automatically populates stock request lines with:
   - All BOM components
   - Calculated quantities based on the BOM quantity
   - Proper unit of measure for each component

<Info>
The auto-fill happens immediately when you select the BOM and quantity.
</Info>

### Example Workflow

**Scenario**: Request components for 10 units of "Laptop Computer"

1. **Create Stock Request Order**
   - Product BOM: Laptop Computer
   - Quantity BOM: 10

2. **System Auto-Fills Lines**:
   ```
   - Laptop Screen    × 10
   - Keyboard         × 10
   - RAM 8GB          × 20 (2 per laptop)
   - SSD 256GB        × 10
   - Battery Pack     × 10
   - Laptop Case      × 10
   ```

3. **Review and Adjust**
   - Review auto-filled lines
   - Manually adjust specific quantities if needed
   - Set expected dates
   - Assign to warehouse location

4. **Confirm Order**
   - Click **Confirm**
   - All component requests are processed

### Updating BOM Quantity

After initial creation, you can adjust the quantity:

1. Open the stock request order
2. Change the **Quantity BOM** field
3. All component line quantities update proportionally

<Note>
Only works for lines that were auto-generated from the BOM. Manual lines remain unchanged.
</Note>

### Clearing BOM Selection

To start over or switch to manual entry:

1. Open the stock request order
2. Use the **Clear BOM** button or action
3. Product BOM and Quantity BOM fields are cleared
4. BOM-generated lines can be removed

## BOM Structure Support

### Single-Level BOMs

```
Product A
  ├─ Component B × 2
  ├─ Component C × 1
  └─ Component D × 4
```

All components are added as direct stock request lines.

### Multi-Level BOMs

```
Product A
  ├─ Subassembly X × 2
  │   ├─ Component B × 3
  │   └─ Component C × 1
  └─ Component D × 4
```

The module handles BOM explosion:
- Subassemblies are expanded
- Final component quantities calculated
- Phantom BOMs processed correctly

<Warning>
Phantom BOM components are expanded automatically. Ensure your BOM structure is correct.
</Warning>

## Troubleshooting

### Lines Not Auto-Filling

<Warning>
If lines don't appear:
</Warning>

- Verify the product has an active BOM
- Check BOM has components defined
- Ensure BOM is valid and not expired
- Verify user has access to BOM data

### Incorrect Quantities

- Review BOM line quantities
- Check unit of measure conversions
- Verify BOM quantity field value
- Look for phantom BOM multipliers

### Cannot Update Quantities

- Ensure order is not confirmed
- Check if lines were manually created vs BOM-generated
- Verify field editability settings
